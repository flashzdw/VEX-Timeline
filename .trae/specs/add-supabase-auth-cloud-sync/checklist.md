# Supabase 用户登录与云端时间轴共享 - 验证清单

## Supabase 基础设施验证
- [x] Supabase 客户端初始化模块 `src/js/supabase.js` 可正常创建客户端实例
- [x] 数据库四张表（users、timelines、timeline_members、records）已创建且字段正确
- [x] RLS 策略已配置：用户只能读取自己的 users 数据
- [x] RLS 策略已配置：timelines 所有者可读写、成员可读、有邀请码的团队时间轴可被查看
- [x] RLS 策略已配置：records 成员可创建和读取、创建者可编辑删除自己的记录
- [x] Storage bucket `record-images` 已创建且可公开访问
- [x] RPC 函数 `register` 和 `login` 可正常调用（register 已修复返回 JWT token）

## 用户认证验证
- [x] 新用户输入未占用用户名可成功注册并自动登录
- [x] 输入已占用用户名注册时提示"用户名已被占用"
- [x] 已有用户输入正确用户名可成功登录
- [x] 输入不存在的用户名登录时提示"用户名不存在"
- [x] 关闭浏览器重新打开后自动恢复登录状态
- [x] 登出功能正常，登出后回到登录页面
- [x] 登录/注册页面 UI 符合 Nothing 设计风格

## 个人时间轴云端同步验证
- [x] 登录用户添加记录后，数据同时保存到 IndexedDB 和 Supabase
- [x] 登录用户编辑记录后，云端数据同步更新
- [x] 登录用户删除记录后，云端数据同步删除
- [x] 在设备 A 添加记录后，设备 B 登录同一账号可看到该记录
- [x] 离线时添加的记录在网络恢复后自动同步到云端

## 赛队时间轴验证

- [x] 所有者可生成并查看邀请码

- [x] 成员可查看和添加赛队时间轴记录
